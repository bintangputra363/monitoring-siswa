Aplikasi Monitoring Siswa

Sebuah sistem informasi berbasis web yang berfungsi sebagai asisten digital untuk memantau kegiatan dan progres siswa.

Aplikasi ini dirancang untuk menggantikan proses manual dalam memantau kegiatan harian siswa (seperti sholat, literasi, dll.) menjadi sebuah sistem digital yang terpusat, efisien, dan dapat diakses secara real-time.

Panduan Instalasi

Ikuti langkah-langkah berikut untuk menjalankan aplikasi di komputer lokal:

  1. Siapkan Server Lokal: Unduh dan install XAMPP. Jalankan modul Apache dan MySQL dari XAMPP Control Panel.
  2. Pindahkan Folder Proyek: Salin folder proyek Anda (`monitoring-siswa`) ke dalam direktori `htdocs` milik XAMPP (biasanya di C:\xampp\htdocs\).
  3. Buat & Impor Database: Buka browser dan akses http://localhost/phpmyadmin. Buat database baru bernama monitor_siswa, lalu pilih database tersebut, klik tab "Import", dan pilih file monitor_siswa (1).sql.
  4. Akses Aplikasi:
    • Login Admin/Guru: http://localhost/monitoring-siswa/Admin/
    • Login Siswa: http://localhost/monitoring-siswa/Starterkit/auth-login.php
  5. Akun default Admin: Username: admin, Password: admin

Struktur Folder

Proyek ini diorganisir ke dalam beberapa direktori utama untuk menjaga kerapian kode:

  • /Admin: Berisi semua file untuk halaman Admin & Guru.
  • /Starterkit: Berisi semua file untuk halaman Siswa.
  • /partials: (Ada di dalam Admin & Starterkit) Berisi file kode yang dipakai berulang, seperti koneksi database (`config.php`), header, dan footer.
  • /assets: Berisi file pendukung seperti gambar, file CSS (untuk styling), dan JavaScript (untuk interaktivitas).
  • monitor_siswa (1).sql: "Cetak biru" atau struktur dari database yang akan kita gunakan.

Bedah Kode: Penjelasan File Penting

Mari kita lihat cara kerja beberapa file kunci dalam aplikasi Anda untuk memahami alur logikanya.

File Koneksi Database: Admin/partials/config.php

File ini adalah jantung aplikasi. Fungsinya adalah untuk membuat jembatan penghubung antara kode PHP dengan database MySQL.


<?php
// Variabel untuk menyimpan detail koneksi
$db_host = "localhost"; // Alamat server database
$db_user = "root";      // Username database
$db_pass = "";          // Password database
$db_name = "monitor_siswa"; // Nama database

// Membuat koneksi ke database menggunakan detail di atas
$conn = new mysqli($db_host, $db_user, $db_pass, $db_name);

// Mengecek apakah koneksi berhasil atau gagal
if ($conn->connect_error) {
    die("Koneksi gagal: " . $conn->connect_error);
}
?>
Logika Menambah Siswa: Admin/partials/add_siswa.php

Saat admin mengisi formulir tambah siswa dan menekan 'Simpan', data tersebut akan diterima dan diproses oleh file ini untuk dimasukkan ke dalam database.


    <?php
    // Memasukkan file koneksi agar bisa terhubung ke database
    require_once 'config.php'; 

    // Mengambil data yang dikirim dari formulir (method POST)
    $nama = $_POST['nama'];
    $nis = $_POST['nis'];
    $kelas_id = $_POST['kelas_id'];

    // Perintah SQL untuk memasukkan (INSERT) data baru ke tabel `siswa`
    $sql_siswa = "INSERT INTO siswa (nama, nis, kelas_id) VALUES ('$nama', '$nis', '$kelas_id')";

    // Menjalankan perintah SQL ke database
    if ($conn->query($sql_siswa) === TRUE) {
        // Jika berhasil, juga masukkan data loginnya ke tabel `users`
        // (di sini ada logika lanjutan untuk INSERT ke tabel users)
        header("location: ../daftar_siswa.php");
    } else {
        // Jika gagal, tampilkan pesan error
        echo "Error: " . $sql_siswa . "<br>" . $conn->error;
    }
    ?>
Logika Checkpoint Siswa: Starterkit/partials/checkpoint.php

Ketika siswa menekan tombol "Checkpoint", file inilah yang bekerja di belakang layar untuk mencatat waktu ke database.


    <?php
    // Memulai session untuk mendapatkan ID siswa yang sedang login
    session_start();
    require_once 'config.php';

    // Mengambil ID siswa (dari session) & ID kegiatan (dari tombol yang diklik)
    $siswa_id = $_SESSION['siswa_id'];
    $kegiatan_id = $_POST['kegiatan_id'];

    // Mengatur zona waktu dan mengambil waktu saat ini
    date_default_timezone_set('Asia/Jakarta');
    $waktu_checkpoint = date('Y-m-d H:i:s');

    // Perintah SQL untuk mencatat checkpoint ke dalam tabel `checkpoint_kegiatan`
    $sql = "INSERT INTO checkpoint_kegiatan (siswa_id, kegiatan_id, waktu_checkpoint, status) 
            VALUES ('$siswa_id', '$kegiatan_id', '$waktu_checkpoint', 'Tepat Waktu')";

    // Menjalankan perintah SQL dan memberikan respon
    if ($conn->query($sql) === TRUE) {
        echo "Checkpoint berhasil!";
    } else {
        echo "Error: " . $conn->error;
    }
    ?>

Skema Database

Berikut rincian setiap tabel dalam database monitor_siswa:

Menyimpan data login (username, password, role) untuk semua pengguna.

Menyimpan data detail profil siswa seperti nama dan NIS.

Menyimpan data detail profil guru.

Menyimpan daftar nama kelas yang ada.

Menyimpan daftar semua kegiatan yang akan dimonitor.

Tabel penting yang menghubungkan data. Contoh: `mapping_siswa_kegiatan` menghubungkan siswa dengan kegiatan yang harus diikutinya.

Mencatat setiap kali siswa melakukan 'checkpoint' (laporan kehadiran) pada sebuah kegiatan. Ini adalah tabel transaksi utama.

Menyimpan informasi file laporan (jika ada) yang diunggah oleh siswa.

Alur Kerja Pengguna

  1. Setup Awal oleh Admin: Admin menyiapkan semua data master: Guru, Siswa, Kelas, dan Kegiatan.
  2. Pemetaan oleh Admin: Admin menentukan kelas mana mengikuti kegiatan apa melalui menu `Mapping Siswa`.
  3. Aktivitas Harian Siswa: Siswa login dan melakukan `Checkpoint` pada kegiatan yang sudah ditentukan.
  4. Monitoring oleh Guru/Admin: Guru dan Admin memantau progres siswa melalui dashboard dan laporan yang tersedia.

Detail Halaman Pengguna

Panel Admin & Guru
  • Dashboard: Grafik dan statistik progres seluruh siswa.
  • Daftar Siswa: Menambah, melihat, mengubah, dan menghapus data siswa.
  • List Guru: Menambah, melihat, mengubah, dan menghapus data guru.
  • Kelas & Kegiatan: Mengelola data master kelas dan kegiatan.
  • Mapping Siswa: Menentukan siswa mana mengikuti kegiatan apa.
Panel Siswa
  • Dashboard: Ringkasan progres pribadi.
  • Kegiatan Siswa: Halaman utama untuk melakukan 'checkpoint' harian.
  • Progress Siswa: Melihat riwayat checkpoint yang sudah dilakukan.
  • Laporan Siswa: Mengunggah file laporan jika diperlukan.
